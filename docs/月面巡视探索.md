## 主题

月面未知环境中，轮式无人平台需要在坡度、障碍、崎岖度、光照、平台能力、任务价值和地图可信度共同约束下，自主选择探索目标并规划局部与全局路径。

## 研究问题逻辑关系

三个研究问题不是并列孤立模块，而是“建模认知 -> 探索决策 -> 路径执行 -> 观测反馈”的闭环关系。

1. 环境-平台-约束模型与可信度是底座，回答世界状态、平台能力和地图可靠性如何表达。
2. 自主探索目标选择是决策层，依赖模型与可信度来判断下一步探索哪里，并需要路径规划反馈目标是否可达、代价是否合理。
3. 全局与局部路径规划是执行层，依赖探索目标和综合代价地图生成可执行路径，执行后的观测再反过来更新地图与可信度。

推荐从最小可用的多层动态地图与可信度地图开始做，然后建立 A* 全局规划基线，再加入探索目标效用函数、观测更新与重规划闭环，最后扩展局部安全走廊、IRIS/GCS 或轨迹优化。

## 工程实现状态

当前本地工程已拆分为三个子项目，并且 `path-planner` 已经取代 `a_gcs_ws-2.0.1` 成为仓库内路径执行评估层：

| 子项目 | 定位 | 当前进展 |
|---|---|---|
| `dev-platform-constraints` | 建模底座 | P0/P1/P2 已形成可运行研究原型，覆盖地图契约、地形特征、平台配置、硬约束、非负代价、可信度更新、覆盖率收益、候选目标和多步序列解释。 |
| `model-explorer` | 探索决策与闭环编排 | 已有 `model-explorer-contract/v1` 加载、目标排序、重规划原因、合成 benchmark、策略训练/评估 smoke 和路径反馈抽象；下一步主要补与 `path-planner` 的真实请求/结果适配。 |
| `path-planner` | 路径规划与执行可行性评估 | 已重建到 Phase 8，包含平台约束 A*、后处理安全走廊、平滑、曲率检查、可跟踪路径、跟踪仿真、固定走廊轨迹优化、执行感知指标和可选 IRIS/区域图诊断。 |

旧的 `a_gcs_ws-2.0.1` 只保留为历史参考和禁用依赖检查，不再作为联调目标。后续接口应围绕 `model-explorer-contract/v1`、`path-planner-request/v1` 和 `path-planner-route/v1` 收敛。

## 下一阶段推进

1. 打通端到端 JSON 联调：由 `dev-platform-constraints` 生成契约报告，`model-explorer` 选择 Top-K 目标并生成 `path-planner-request/v1`，`path-planner` 输出 `path-planner-route/v1`，再把可达性、代价、安全和失败原因回写到目标排序。
2. 固化 `model-explorer` 的路径反馈策略：先使用合同代价和 CLI JSON 适配，明确何时因为不可达、高风险、跟踪安全失败或优化 fallback 触发重规划。
3. 扩展半真实数据验证：以 `dev-platform-constraints` 的 `.npz` 验证地图为起点，补齐多地形、多光照、低可信风险带和障碍密集场景的端到端回归。
4. 推进 `path-planner` 的生产化边界：保持当前 A* 和固定走廊优化为可靠 fallback，逐步验证 Drake/IRIS 区域图质量，再决定是否进入完整 GCS 轨迹优化。
5. 建立统一验收指标：覆盖率提升、路径代价、低可信高风险暴露、不可达率、重规划次数、跟踪误差代理、优化 fallback 比例和报告可复现性应成为下一轮实验表格的核心列。

当前首版 `model-explorer` 适配器已能通过 `path_planner_route` 后端生成 `path-planner-request/v1`，并通过 `path-planner` CLI 子进程消费 `path-planner-route/v1`。由于 `model-explorer-contract/v1` 稳定字段暂不包含完整 `cost` 和 `passable_mask` 数组，合成 smoke 可使用 open-grid fallback；半真实验证必须补充来自 `dev-platform-constraints` 的真实代价图和硬约束掩膜。

半真实闭环的当前推进方式是由 `dev-platform-constraints` 导出成对 JSON：`model-explorer-contract/v1` 保持候选目标与观测摘要，`path-planner-sidecar/v1` 补充 `cost`、`passable_mask` 和 terrain layers。`model-explorer` 使用 `path_planner_route + path_planner_sidecar` 配置批量评估 Top-K 目标，并把不可达、路径代价、失败原因、跟踪安全和优化 fallback 写入路径反馈摘要。IRIS/region graph 当前只作为诊断特征；只有当它能稳定解释路径失败或高风险暴露后，才进入完整 GCS/Drake 替换评估。

下一步实验入口已收敛为 `model_explorer path-feedback run <manifest>`。该入口记录路径反馈前后的目标变化、`open_grid_fallback_used`、路径失败数、重规划数、跟踪安全违规数、轨迹优化 fallback 数、region graph 断连数和单位路径代价覆盖收益。半真实实验结论必须基于 sidecar 的真实 `cost/passable_mask`，不能基于 open-grid fallback。
